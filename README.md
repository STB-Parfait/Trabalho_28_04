# Agenda API

College project for the ISS A2 assignment. The system is a REST API for managing an agenda with contacts and appointments, implemented in pure Java using the same style shown in `iss-aula05.pdf`: `HttpServer`, HTTP endpoints, and JSON responses.

## What The Project Does

The API manages two related entities:

- `Contact`: a person with name, email, and phone.
- `Appointment`: an agenda item linked to one contact.

It includes real business rules beyond basic CRUD:

- Required field validation.
- Email format validation.
- Unique contact email.
- Appointment start date must be before end date.
- Appointment contact must exist.
- Appointment time overlap is blocked for the same contact.
- Appointment duration is calculated in minutes.
- Appointment status is classified as `SCHEDULED`, `TODAY`, `PAST`, or `CANCELED`.
- The report endpoint consumes internal HTTP endpoints using Java `HttpClient`, demonstrating service integration.

## Requirements

- Docker and Docker Compose.
- Java 21 and Maven, only if running outside Docker.

## Running With Docker

Create your local environment file if you want to customize ports or credentials:

```bash
cp .env.example .env
```

Start the database, phpMyAdmin, and Java API:

```bash
docker compose up --build
```

Services:

- API: `http://localhost:8000`
- Swagger UI: `http://localhost:8000/swagger`
- OpenAPI JSON: `http://localhost:8000/openapi.json`
- phpMyAdmin: `http://localhost:8081`
- MySQL: `localhost:3306`

Database credentials:

- Database: `agenda_db`
- User: `agenda_user`
- Password: `agenda_password`
- Root password: `root_password`

The database schema is created from [db/schema.sql](db/schema.sql).

Docker Compose uses the variables documented in [.env.example](.env.example). If no `.env` file exists, the project still runs with the default values shown above.

## Running Locally Without Docker

Start MySQL first, then set the database environment variables if needed:

```bash
export DB_HOST=localhost
export DB_PORT=3306
export DB_NAME=agenda_db
export DB_USER=agenda_user
export DB_PASSWORD=agenda_password
export APP_PORT=8000
mvn package
java -jar target/agenda-api.jar
```

## Endpoints

Interactive API documentation is available through Swagger UI:

```text
http://localhost:8000/swagger
```

The raw OpenAPI specification is available at:

```text
http://localhost:8000/openapi.json
```

### Contacts

`GET /contacts`

Lists all contacts.

`GET /contacts/{id}`

Returns one contact.

`POST /contacts`

Creates a contact.

```json
{
  "name": "Maria Silva",
  "email": "maria@example.com",
  "phone": "(11) 99999-0000"
}
```

`PUT /contacts/{id}`

Updates a contact.

`DELETE /contacts/{id}`

Deletes a contact and its appointments.

### Appointments

`GET /appointments`

Lists all appointments.

`GET /appointments/{id}`

Returns one appointment.

`POST /appointments`

Creates an appointment.

```json
{
  "contactId": 1,
  "title": "Project meeting",
  "description": "Discuss ISS A2 implementation",
  "startsAt": "2026-04-28T14:00:00",
  "endsAt": "2026-04-28T15:00:00"
}
```

`PUT /appointments/{id}`

Updates an appointment. To cancel it, send `"status": "CANCELED"`.

`DELETE /appointments/{id}`

Deletes an appointment.

### Reports

`GET /reports/agenda-summary`

Returns a summary generated by consuming the internal `/contacts` and `/appointments` endpoints through HTTP.

Example response:

```json
{
  "totalContacts": 1,
  "totalAppointments": 2,
  "scheduledAppointments": 1,
  "todayAppointments": 0,
  "pastAppointments": 1,
  "canceledAppointments": 0
}
```

## Code Organization

- `App.java`: starts the HTTP server and registers routes.
- `controllers`: route handlers using `HttpServer`.
- `services`: business rules and validation.
- `repositories`: JDBC database access.
- `models`: domain classes and response models.
- `exceptions`: custom application exceptions.
- `config`: database and JSON configuration.
- `tests/ApiSmokeTest.java`: small Java HTTP test client using `HttpURLConnection`.

This structure separates HTTP, business rules, data access, and models, as required by the assignment.

## Quick Manual Test

After `docker compose up --build`, create a contact:

```bash
curl -X POST http://localhost:8000/contacts \
  -H "Content-Type: application/json" \
  -d '{"name":"Maria Silva","email":"maria@example.com","phone":"(11) 99999-0000"}'
```

Create an appointment:

```bash
curl -X POST http://localhost:8000/appointments \
  -H "Content-Type: application/json" \
  -d '{"contactId":1,"title":"Project meeting","description":"Discuss project","startsAt":"2026-04-28T14:00:00","endsAt":"2026-04-28T15:00:00"}'
```

Generate the summary report:

```bash
curl http://localhost:8000/reports/agenda-summary
```
